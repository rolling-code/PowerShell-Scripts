Consent Phishing Simulation (Multi-Tenant Demo)

⚠️ This was a simulation – no credentials were captured.

You interacted with a multi-tenant application that requested access via Microsoft’s real sign-in page.

What this demonstrates

  • Attackers register apps in their own tenant and make them multi-tenant.
  • They ask for dangerous permissions like Mail.Read, Files.Read, and offline_access.
  • If consent is granted, attackers can access data and persist using refresh tokens until consent is revoked.

How to protect yourself

  • Verify the publisher and look for verified publisher badges.
  • Scrutinize the permissions and ask: do they match the task?
  • If uncertain, cancel and report to IT Security.
