⚠️ This was a phishing simulation – no credentials were captured.

You clicked a link and did not sign in, well done. This exercise demonstrates how attackers exploit trusted urls to trick users.

What happened?

  • The sign-in page looked legitimate because it was Microsoft’s real login page.

How to protect yourself:

  • Always verify the URL in the address bar before entering credentials.
  • Always verify the AIM Image is present before entering credentials.
  • If something feels off, stop and report it to IT Security.
